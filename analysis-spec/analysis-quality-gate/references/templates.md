# 分析质量门禁 — 模板集

> 本文件包含阶段一到阶段三使用的结构化模板。SKILL.md 中引用此文件时按需读取对应章节。

---

## 1. AskQuestion 预检模板

```
问题1: 这份分析的核心受众和决策目的？
  选项: [高管汇报(简洁+行动导向)] / [团队内部(可深入技术细节)] / [跨部门协作(兼顾背景介绍)] / [自定义]

问题2: 分析时间窗口和对比基期？
  选项: [最近1个月 vs 上月] / [最近3个月 vs 去年同期] / [最近12个月趋势] / [自定义]

问题3: 所有数据是否已准备好一次性提供？
  选项: [是,全部就绪] / [还有部分待补充,先说明缺什么]

问题4: 是否有已知的重大事件/变动需在分析中标注或排除？
  选项: [无] / [有,请说明]

问题5: 期望的交付物形式？
  选项: [MD+HTML完整报告] / [一页纸摘要] / [仅数据表+结论] / [自定义]

问题6: 验收标准（什么样的报告你会满意）？
  选项: [三视角质检框架(参谋长+麦肯锡+CEO)] / [给我看看框架我来确认] / [我自定义标准]
```

## 5. 三视角质检评分模板

每个检查项为**通过/不通过**二元判定，每视角 5 项，计通过数（0-5）。

```
三视角质检：
├── 参谋长（决策可用性）：_/5
│   ├── 速览 + So What：报告顶部有30秒速览，每节有决策含义？ [通过/不通过]
│   ├── 行动建议：有≥3条带优先级(P0/P1)、时间窗口、预期效果的建议？ [通过/不通过]
│   ├── 前瞻推演：有"不做vs做了"的情景对比？ [通过/不通过]
│   ├── 表述亲和：非数据人员能看懂核心结论？ [通过/不通过]
│   └── 量级感：数字翻译为资源/时间/ROI含义？ [通过/不通过]
│
├── 麦肯锡（方法论严谨）：_/5
│   ├── 假设透明：每个关键假设有依据说明？ [通过/不通过]
│   ├── 区间表述：结论用区间而非点估计？ [通过/不通过]
│   ├── 敏感性分析：关键假设±30%影响有定量展示？ [通过/不通过]
│   ├── 盲区定量：已知盲区有粗估(保守/中性/激进)？ [通过/不通过]
│   └── 交叉校验：不同方法/数据源有验证收敛性？ [通过/不通过]
│
└── CEO（战略前瞻）：_/5
    ├── 趋势方向：说明趋势往哪走、在加速还是减速？ [通过/不通过]
    ├── 威胁排序：按影响×紧迫度排序而非等权罗列？ [通过/不通过]
    ├── 资源含义：数字翻译为"值得投入多少资源"？ [通过/不通过]
    ├── 时间窗口：明确"什么时候必须行动"？ [通过/不通过]
    └── 一句话版本：≤30字传达核心信息？ [通过/不通过]

关键改进项（本次迭代修正的 top 3）：
1. [具体改进]
2. [具体改进]
3. [具体改进]

遗留短板（受限于数据/方法无法解决的问题）：
- [坦诚列出]
```

### 迭代规则

| 评分区间 | 动作 |
|---------|------|
| 任一视角 < 3/5 | 修正不通过项后重评 |
| 3~4/5 | 建议迭代，标注具体改进方向 |
| 5/5 | 通过，标注遗留短板即可 |

**迭代上限：最多 2 轮。** 超过 2 轮仍有不通过项，标注遗留短板后直接输出。
